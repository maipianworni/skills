# Cross-Skill Workflows [LEAF NODE] — 模板

> 本模板用于生成 `cross-cutting/SKILL.md`。cross-cutting 是 Multi-Skill 树中的**工作流组合器**，兼具两个职责：
> 1. **预定义工作流字典** — 覆盖高频跨 skill 组合场景
> 2. **自定义工作流回退** — 当用户意图不匹配任何预定义工作流时，指导 agent 自行分解任务、逐个 skill 路由

## 文件结构

生成的 `cross-cutting/SKILL.md` 必须包含以下段落（按顺序）：

```
# Cross-Skill Workflows [LEAF NODE]

## 目标 Skills
## 何时使用
## 何时不使用
## 预定义工作流                  ← 预定义工作流表格 + bash 示例
## 自定义工作流组合              ← 【关键】回退机制，必须包含
## Constraints
```

## 各段落生成规则

### 目标 Skills
列出树中所有 skill 名称，用 ` + ` 连接。

### 何时使用 / 何时不使用
```
## 何时使用
- 跨 skill 工作流, pipeline, 批量处理, 跨领域
- 需要组合不同 skill 的能力完成复杂任务
- 从一个 skill 的输出作为另一个 skill 的输入

## 何时不使用
- 只涉及一个 skill → 路由到对应 skill 的子树
- 简单的单步任务 → 直接使用对应叶节点
```

### 预定义工作流
为每对/每组有自然协作关系的 skill 生成一行：

```markdown
| 工作流 | 涉及 Skills | 说明 |
|--------|-----------|------|
| {工作流名称} | {skill_a} + {skill_b} | {简要描述数据流和步骤} |
```

为每个预定义工作流提供示例：

```markdown
### 工作流 1: {名称}

**触发条件**: {什么样的用户请求会触发}

**步骤**:
1. 使用 {skill_a} 完成 {子任务描述} → 路由到 `{skill_a}/{module}/SKILL.md`
2. 将上一步结果传递给 {skill_b}
3. 使用 {skill_b} 完成 {子任务描述} → 路由到 `{skill_b}/{module}/SKILL.md`
```

### 自定义工作流组合（必须包含）
**这是 cross-cutting 的核心能力，不可省略。**

```markdown
## 自定义工作流组合（无预定义匹配时的回退机制）

当用户意图涉及多个 skill 但不在上方预定义工作流表中时，按以下流程自行组合：

### 步骤 1: 识别涉及的 Skills
从用户意图中提取每个子任务及其对应的 skill（参照 ROOT.md Phase 1 的关键词信号）。

### 步骤 2: 确定执行顺序
按数据流排列：哪个 skill 的输出是下一个 skill 的输入。
- 有明确先后依赖 → 串行执行
- 无依赖关系 → 并行执行

### 步骤 3: 逐个 Skill 路由到具体叶节点
对每个子任务，回到对应 skill 的 ROUTER.md → leaf SKILL.md，找到精确执行指令。不要猜测指令，必须经过路由树查找。

### 步骤 4: 串联执行
按步骤 2 确定的顺序，依次读取并执行每个叶节点的指令。上一步的输出作为下一步的输入。
```

生成时需**根据树中实际 skill 举一个具体示例**，展示完整的分解→路由→执行过程。

## 检查清单

- [ ] 目标 Skills 列出树中所有 skill（无遗漏）
- [ ] 预定义工作流表每行涉及 ≥2 个 skill
- [ ] 每个预定义工作流有对应的步骤示例
- [ ] **自定义工作流组合段落存在且包含 4 个步骤**
- [ ] 自定义工作流示例使用了树中实际存在的 skill
